## 1. 产品概述

### 1.1 功能特性

| 模块 | 功能说明 |
|------|----------|
| **多账号管理** | 账号新增、编辑、删除、启动、停止；扫码登录（QQ/微信）；离线自动删除与推送通知 |
| **农场自动化** | 收获、种植、浇水、除草、除虫、铲除、土地升级；仓库自动出售果实 |
| **好友互动** | 自动偷菜 / 帮忙 / 捣乱；好友黑名单；静默时段；60 秒防偷抢收保护 |
| **任务系统** | 自动检查并领取任务奖励 |
| **Web 面板** | 概览、农场、背包、好友、分析、账号、设置、帮助中心；实时日志；深色/浅色主题 |

### 1.2 技术栈与环境要求

| 项目 | 要求 |
|------|------|
| Node.js | 20+（源码运行） |
| MySQL | 8.0+（可选） |
| Redis | 6.0+（可选） |
| Docker | Engine 20+、Compose v2+（Docker 部署） |
| 二进制版 | 无需 Node.js，直接运行 |

## 2. 快速开始

### 2.1 最快上手：Docker 一键部署（推荐）

**x86_64（Intel/AMD 云服务器）：**
```bash
curl -O https://raw.githubusercontent.com/smdk000/qq-farm-ui-pro-max/main/scripts/deploy-x86.sh && chmod +x deploy-x86.sh && ./deploy-x86.sh
```

**ARM64（甲骨文 ARM、树莓派、Apple Silicon）：**
```bash
curl -O https://raw.githubusercontent.com/smdk000/qq-farm-ui-pro-max/main/scripts/deploy-arm.sh && chmod +x deploy-arm.sh && ./deploy-arm.sh
```

- 约 2 分钟内完成，日志出现 `✅ MySQL 核心表结构自动初始化完成` 即成功
- 访问：`http://localhost:3080`
- 默认账号：`admin` / `qq007qq008`（**建议部署后立即修改密码**）

### 2.2 本地开发：源码运行

```bash
# 1. 安装 Node.js 20+，启用 pnpm
corepack enable

# 2. 安装依赖并构建
pnpm install
pnpm build:web

# 3. 启动
pnpm dev:core
```

访问：`http://localhost:3000`

### 2.3 一键开发脚本（ARM Mac）

项目根目录提供 `dev.sh`，可一键关闭旧进程、编译前端、启动后端：

```bash
./dev.sh
```

## 3. 部署方式详解

### 3.1 方式一：Docker 一键部署

见 [2.1 最快上手](#21-最快上手docker-一键部署推荐)。

### 3.2 方式二：Docker Compose（生产环境）

```bash
curl -O https://raw.githubusercontent.com/smdk000/qq-farm-ui-pro-max/main/docker-compose.prod.yml
curl -o .env https://raw.githubusercontent.com/smdk000/qq-farm-ui-pro-max/main/.env.example
# 编辑 .env 修改 ADMIN_PASSWORD 等
docker-compose -f docker-compose.prod.yml up -d
```

### 3.3 方式三：单容器运行

```bash
docker run -d \
  --name qq-farm-bot-ui \
  --restart unless-stopped \
  -p 3080:3000 \
  -v ./data:/app/core/data \
  -v ./logs:/app/logs \
  -v ./backup:/app/core/backup \
  -e ADMIN_PASSWORD=qq007qq008 \
  -e TZ=Asia/Shanghai \
  smdk000/qq-farm-bot-ui:latest
```

### 3.4 方式四：二进制版（无需 Node.js）

从 [GitHub Releases](https://github.com/smdk000/qq-farm-ui-pro-max/releases) 下载对应平台可执行文件，直接运行。

| 平台 | 文件名 |
|------|--------|
| Windows x64 | `qq-farm-bot-win-x64.exe` |
| Linux x64 | `qq-farm-bot-linux-x64` |
| macOS Intel | `qq-farm-bot-macos-x64` |
| macOS Apple Silicon | `qq-farm-bot-macos-arm64` |

访问 `http://localhost:3000`，默认 `admin` / `admin`。

### 3.5 Docker 环境变量说明

| 环境变量 | 说明 | 默认值 |

## 4. 账号管理

### 4.1 添加账号

1. 进入「账号」页面
2. 点击「添加账号」
3. 选择登录方式：
   - **扫码登录**：支持 QQ 与微信，扫码后自动获取凭证
   - **手动输入**：输入 QID 和密码（需配合验证码）

### 4.2 账号操作

| 操作 | 说明 |
|------|------|
| 启动 | 开始自动化运行 |
| 停止 | 暂停自动化 |
| 编辑 | 修改备注等信息 |
| 删除 | 移除账号（谨慎操作） |

### 4.3 账号状态说明

- **在线**：正常运行
- **离线**：掉线或未启动
- **被踢下线**：账号被官方检测，将自动删除
- **连续离线超时**：超过设定时间未恢复，将自动删除

### 4.4 离线推送通知

支持 Bark、自定义 Webhook 等渠道，账号离线时推送提醒。在「设置」→「离线通知」中配置。

## 6. 设置详解

> 完整设置说明见 [docs/guides/SETTINGS_GUIDE.md](guides/SETTINGS_GUIDE.md)，此处为要点摘要。

### 6.1 巡查间隔设置

| 设置项 | 推荐值 | 说明 |
|--------|--------|------|
| 农场巡查最小 | 180 秒 | 检查自己农场的间隔 |
| 农场巡查最大 | 300 秒 | 3-5 分钟为平衡配置 |
| 好友巡查最小 | 600 秒 | 检查好友农场的间隔 |
| 好友巡查最大 | 900 秒 | 10-15 分钟为平衡配置 |

⚠️ **不要设置过短**：2 秒、10 秒等过于频繁，容易被检测。

### 6.2 静默时段

- **作用**：在指定时间段内暂停所有好友互动（偷菜、帮忙、捣乱）
- **推荐**：`23:00 - 07:00`（8 小时）
- **建议**：务必开启，模拟真人作息

### 6.3 自动控制功能

| 功能 | 推荐 | 说明 |
|------|------|------|
| 自动种植收获 | ✅ 开启 | 核心总开关 |
| 自动好友互动 | ✅ 开启 | 偷菜/帮忙/捣乱总开关 |
| 自动卖果实 | ✅ 开启 | 收获后自动出售 |
| 自动升级土地 | 按需 | 金币充足时开启 |
| 推送触发巡田 | ✅ 开启 | 收到推送立即巡查 |
| 自动捣乱 | ❌ 关闭 | 收益低、易引起不满 |

### 6.4 施肥策略

```
普通 + 有机 ✅ 推荐（收益最高）
仅普通化肥（节省有机肥）
仅有机化肥（有机肥充足时）
不施肥（暂时不需要增益）
```

### 6.5 推荐配置方案

**平衡配置（推荐）**：
```
农场巡查：3-5 分钟 (180-300 秒)
好友巡查：10-15 分钟 (600-900 秒)
静默时段：23:00-07:00
施肥策略：普通 + 有机
自动偷菜：开启
自动帮忙：开启
自动捣乱：关闭
经验上限停止帮忙：开启
```

## 7. 偷菜过滤配置

### 7.1 配置入口

1. 进入「设置」
2. 选择账号
3. 点击「偷菜过滤设置」

### 7.2 按作物过滤

| 模式 | 说明 |
|------|------|
| **黑名单** | 不偷选中的蔬菜 |
| **白名单** | 只偷选中的蔬菜 |

### 7.3 按好友过滤

| 模式 | 说明 |
|------|------|
| **黑名单** | 不偷列表中的好友 |
| **白名单** | 只偷列表中的好友 |

### 7.4 使用建议

- **新手**：先不开启过滤，运行一段时间后再配置
- **进阶**：黑名单过滤低价值作物、亲友账号
- **专业**：白名单只偷高价值作物和「肥羊」好友

```bash
tar -czf farm-bot-backup-$(date +%Y%m%d).tar.gz ./data
```

## 10. 故障排查

> 完整故障排查见 [docs/guides/TROUBLESHOOTING.md](guides/TROUBLESHOOTING.md)。

### 10.1 服务无法启动

```bash
# 检查 Node.js 版本
node -v  # 必须 >= v20

# 检查端口占用
lsof -i :3000   # Linux/macOS
netstat -ano | findstr :3000  # Windows

# 查看错误日志
tail -f logs/app.log

# 重新安装依赖
pnpm install --force
```

### 10.2 账号无法登录

1. 检查网络连接
2. 清除浏览器缓存
3. 重新生成二维码
4. 检查系统时间是否准确

### 10.3 账号频繁掉线

- 降低自动化频率（巡查间隔）
- 添加静默时段
- 检查网络稳定性

### 10.4 数据库错误

```bash
# 停止服务 → 备份 → 检查完整性
sqlite3 data/farm-bot.db "PRAGMA integrity_check;"
# 若损坏，从备份恢复
```

### 10.5 常见错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| `AUTH_FAILED` | 认证失败 | 重新登录 |
| `NETWORK_ERROR` | 网络错误 | 检查网络 |
| `DATABASE_LOCKED` | 数据库锁 | 重启服务 |
| `ACCOUNT_OFFLINE` | 账号离线 | 检查账号状态 |

## 11. 常见问题 FAQ

### Q1: 如何添加账号？

进入「账号」页面 → 添加账号，支持扫码或手动输入 QID/密码。

### Q2: 如何配置偷菜过滤？

见 [7. 偷菜过滤配置](#7-偷菜过滤配置)。

### Q3: 为什么开启了自动偷菜但不偷？

检查：① 是否开启「自动好友互动」；② 是否在静默时段；③ 是否设置了过滤；④ 好友是否有成熟作物；⑤ 账号是否在线。

### Q4: 配置修改后多久生效？

**立即生效**，下次巡查即按新配置执行，无需重启。

### Q5: Docker 镜像拉取失败？

确认使用 `smdk000/qq-farm-bot-ui:latest`，执行 `docker login`。

### Q6: 端口被占用？

检查 `lsof -i :3080`，或修改 `PORT` 环境变量。

### Q7: 默认密码是什么？

- Docker 部署：`admin` / `qq007qq008`
- 二进制版：`admin` / `admin`
- **建议部署后立即修改**

## 12. 最佳实践与攻略

### 12.1 新手入门（第 1 周）

1. 使用**保守配置**，先让机器人跑起来
2. 观察运行情况，查看日志是否正常
3. 不要开启过滤，先自由运行
4. 每天检查几次，确保账号正常

### 12.2 进阶优化（第 2-4 周）

1. 切换到**平衡配置**，提升收益
2. 配置偷菜过滤，过滤低价值作物
3. 配置好友过滤，过滤亲友账号
4. 根据日志调整巡查间隔

### 12.3 专业调优（1 个月后）

1. 分析收益数据，找到最优配置
2. 精细化过滤，只偷高价值目标
3. 多账号协同，最大化总收益
4. 定期调整策略，适应游戏变化

### 12.4 安全第一

1. **巡查间隔**：农场 ≥2 分钟，好友 ≥5 分钟
2. **静默时段**：建议至少 6 小时
3. **谨慎捣乱**：容易引起好友不满
4. **定期检查**：确保账号正常运行

### 12.5 配置检查清单

保存配置前请确认：

- [ ] 农场巡查间隔：3-5 分钟
- [ ] 好友巡查间隔：10-15 分钟
- [ ] 静默时段：23:00-07:00
- [ ] 自动种植收获：开启
- [ ] 自动好友互动：开启
- [ ] 自动卖果实：开启
- [ ] 推送触发巡田：开启
- [ ] 经验上限停止帮忙：开启
- [ ] 自动捣乱：关闭（推荐）
